Q1. ログに ±0.01 秒程度の補正が頻繁に表示されます。異常ですか？

A. 異常ではありません。設計どおりの正常動作です。

Windows のシステム時刻は約 10ms 単位で量子化されており、
ChronoGPS が GNSS 基準で評価すると、
±0.01 秒前後の補正が繰り返し見えることがあります。

これは GNSS が不安定なのではなく、Windows 側の特性です。
補正後は常に数ミリ秒以内に収束しており、
FT8 / FT4 の要求精度（±1秒）に対しては十分すぎる精度です。

Q2. 「Instant Sync を一度行えば十分」とは、同期をオフにしてよいという意味ですか？

A. いいえ。その意味ではありません。

「十分」というのは、

OS のシステム時刻を書き換える操作を、頻繁に繰り返す必要がない

という意味です。

GPS 受信自体は、
表示・監視・検証目的で 継続していても問題ありません。

Instant Sync は
適切な代表値で一度校正するためのモードであり、
毎秒追従する同期方式ではありません。

Q3. Weak Sync（Interval Sync）を常用したほうが精度は上がりますか？

A. 上がりません。用途が異なります。

Weak Sync は、

長時間稼働時のドリフト監視

GNSS 受信状態の確認

動作検証・診断

を目的とした 監視中心の同期モードです。

FT8 / FT4 の通常運用では
Instant Sync が最も適切で、推奨される使い方です。

Q4. ログに少しずつ一方向にずれていく表示があります。同期に失敗していますか？

A. 失敗ではありません。正常な挙動です。

-0.03s → -0.05s のような緩やかな変化は、
PC のシステムクロックが持つ 自然なドリフトです。

ChronoGPS は、

閾値内 → あえて補正しない

閾値超過 → 必要なときだけ補正する

という保守的な判断を行い、
GNSS の微小なジッタを OS 時刻に注入しないことを優先しています。

Q5. NTP と GPS は同時に使うべきですか？

A. 目的によります。ChronoGPS は競合させません。

GPS / GNSS：絶対的な UTC 基準

NTP：ネットワーク経由の時刻検証・補助

ChronoGPS は
どちらかを主、もう一方を検証用として使える設計です。

FT8 / FT4 運用では
GPS（Instant Sync）単独で十分な精度が得られます。
